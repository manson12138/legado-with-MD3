# M3 普通规则固定样本与预期结果

以下样本用于用户补充自动化测试或临时调试入口。本文只记录预期，没有宣称已经由 AI 运行。

## JSONPath

输入：

```json
{"books":[{"name":"甲","url":"/a"},{"name":"乙","url":"/b"}]}
```

- 列表规则：`@Json:$.books[*]`
- 字段规则：`$.name`、`$.url`
- 预期：2 项；名称依次为“甲”“乙”；地址基于最终响应 URL 转绝对地址。

## CSS

输入：

```html
<ul><li><a href="/a"><span>甲</span></a></li><li><a href="/b">乙</a></li></ul>
```

- 列表规则：`@CSS:li`
- 名称规则：`a@text`
- 地址规则：`a@href`
- 预期：2 项；名称为“甲”“乙”；地址为 `/a`、`/b` 对应的绝对 URL。

## XPath

输入使用上一个 HTML。

- 列表规则：`@XPath://li`
- 名称规则：`@XPath:.//a/text()`
- 地址规则：`@XPath:.//a/@href`
- 预期：2 项，字段与 CSS 样本一致。
- 注意：HTML 解析器可能自动补全 `tbody` 等节点，真实书源必须与 Android JSoup 结果逐项比较。

## Regex

输入：`<book name="甲" url="/a"/><book name="乙" url="/b"/>`

- 列表规则：`:<book[^>]+/>`
- 名称规则：`:name="([^"]+)"`
- 地址规则：`:url="([^"]+)"`
- 预期：2 项，捕获组分别为“甲”/“乙”和 `/a`/`/b`。
- 差异门禁：Dart 与 Java 正则的回溯、Unicode 类和部分高级结构可能不同，失败必须报告解析异常，不能返回空列表掩盖。

## 字符串替换和组合

- 规则：`@CSS:.name@text##^书名[:：]\\s*##`
- 输入：`<span class="name">书名：甲</span>`
- 预期：“甲”。
- `A||B`：使用第一组非空结果。
- `A&&B`：顺序拼接两组结果。
- `A%%B`：按索引交错两组结果。

## 四段固定链路

| 阶段 | 固定输入 | 关键预期 |
|---|---|---|
| 搜索 | JSONPath 或 CSS 书籍列表 | 非空书名才产出；按绝对 `bookUrl` 去重；`bookList` 以 `-` 开头时反转 |
| 详情 | 单书详情 HTML | `canReName` 控制书名作者覆盖；封面和目录地址绝对化 |
| 目录 | 章节列表 HTML | 空章节 URL 按 Android 规则回退；标题为空跳过；分页防循环；最终索引连续 |
| 正文 | 正文 HTML | 块级标签转换行；空正文与规则异常区分；分页防循环并按顺序合并 |

## 用户验收建议

1. 执行 `cd flutter_app && flutter pub get`。
2. 执行 `flutter analyze`。
3. 为上述五类规则各添加一个固定测试，并执行 `flutter test`。
4. 选择不含 JS/WebView/DNS/代理的普通书源，依次确认搜索、详情、目录、正文关键字段。
5. 用 GBK 页面、重定向相对链接、持久 Cookie、会话 Cookie、ZIP 响应各验证一次。
6. 验证取消、总超时、DNS、TLS、HTTP 状态、解码失败和合法空匹配能被分别识别。
7. 将命令输出和真实书源差异反馈后，再决定是否更新为 `ANDROID_READY`。
