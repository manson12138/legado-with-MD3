## 已实现

- M7 `/reader` 路由正式连接 `ReaderRoute`，入口参数继续使用稳定 `bookUrl`，缺失或空参数在路由边界拒绝。
- `ReaderUiState`、Intent、Effect、ViewModel、Route 和纯 `ReaderScreen`；正文加载、进度、设置、书签和平台窗口操作均不写入 Widget 业务回调。
- `ReadBookCoordinator` 对应 Android `ReadBook` 第一批职责：当前章加载、取消、内存 LRU、原始正文持久缓存、后台净化分块及前后章预加载。
- 当前章请求会取消旧当前章与全部低优先级预加载；ViewModel 和协调器分别使用加载世代，旧 Future 即使晚返回也不能覆盖新章节。
- 正文缓存使用 SHA-256 隐藏原始 URL，保存书源解析后的原始正文七天；强制重试可绕过缓存，替换开关变化可从原始缓存重新处理。
- 正文换行净化、重复标题移除、文本/正则替换和约 1200 字符分块在专用 isolate 执行；处理任务根据规则超时声明设置 3～12 秒上限，超时会杀死 isolate。
- 正文使用 `ListView.builder` 惰性构建分块，稳定 key 由章节 URL 与块序号组成，未把整章交给单个超大 `Text`。
- 阅读位置同时保存到 Android 兼容的 `books` 进度字段和独立稳定锚点；锚点包含章节 URL、章节索引、章节内字符位置及后续 80 字摘要，不保存滚动像素。
- 再次打开时优先匹配章节 URL；正文变化后先在旧位置附近查找摘要，再回退受控字符位置。字体、行距、段距和边距变化后重新将字符锚点投影到当前布局。
- 滚动状态更新 250ms 节流，持久进度 1200ms 节流；切章、进入后台和正常退出都会立即保存。
- 支持上一章、下一章、完整目录跳转、错误强制重试和章末连续阅读；卷标题自动跳过且不能直接打开。
- 支持字号、行距、段距、左右边距、纸张/护眼/深色配色、替换规则开关和屏幕常亮开关，单书配置保存在通用缓存表。
- 支持当前位置添加书签、书签列表、书签跳转和删除；仍保留 Android 以书名与作者关联书签的字段语义。
- 当前章完成后按“下一章、上一章”顺序预加载；当前请求优先，预加载失败静默丢弃，内存处理结果最多保留三章。
- Flutter `SystemChrome` 负责沉浸系统栏；Android MethodChannel 使用 `FLAG_KEEP_SCREEN_ON`，iOS 使用 `isIdleTimerDisabled`，两端退出时恢复进入阅读器前状态。
- P0 阅读 UI 已从普通 AppBar/BottomAppBar 改为全屏正文 + `ReaderMenuOverlay` 浮层；顶部保留返回、章节标题、刷新当前章、整书换源和更多菜单，底部保留章节进度滑杆、章节数、章节内百分比、上一章、目录、书签、设置和下一章。
- 章节进度滑杆只在本地维护拖动草稿，松手后查找最近可阅读章节并复用 `OpenReaderChapterIntent`；卷标题不会被直接打开。
- P1 显示设置已拆成“样式 / 文字 / 间距 / 系统”四个页签；新增上下边距、字距、字重和斜体字段，并同步到显示配置缓存。
- 连续滚动正文和分页正文都已消费 P1 新增排版字段，配置变化后继续通过稳定字符锚点重新投影阅读位置。
- P2 目录面板打开时按当前章节初始定位，并支持显示/隐藏卷标题；卷标题仍不会被直接打开。
- 顶部更多菜单新增正文搜索、替换统计、刷新后续章节和刷新全部章节入口；当前章刷新继续保留在顶栏。
- 搜索面板支持当前章/整书范围切换；整书搜索按目录顺序加载可阅读章节，结果支持计数、上一条/下一条、点击跨章节跳转和稳定字符锚点保存。
- 书签列表新增编辑入口，删除操作增加二次确认；书签编辑面板支持修改备注，空备注回退到创建书签时的正文摘要；书签导出会把 Markdown 文本复制到剪贴板。
- 替换统计面板展示当前章节替换开关、生效数量、正文是否来自缓存和当前书可用的完整正文替换规则列表；规则编辑仍留给后续规则页面迁移。
- P3/P4 可落地项新增覆盖/无动画/滑动翻页策略、页眉页脚、底部工具文字、文字阴影和文字下划线配置，并同步到显示配置缓存；无论新配置还是旧缓存，首次进入新版阅读器都会迁移为左右分页 + 覆盖翻页，用户之后手动选择的模式继续保留。
- 连续滚动正文和分页正文已消费页眉页脚、文字阴影和下划线；覆盖模式支持左右跟手拖动和左右区域点击，翻下一页时当前纸张向左移走、翻上一页时上一页从左侧覆盖回来。
- 相邻章节切换会保留当前页直到目标章节加载完成；下一章首页在旧章末页下方衔接，上一章末页从左侧覆盖回来，不再清空正文并刷新整张列表。
- 分页引擎已从“整章子字符串二分 + 段落换行回退”改为按章节标题、正文段落和 `TextPainter` 真实排版行逐行装页；第一页显示独立大号章节标题，正文段距和两个全角空格首行缩进参与分页。
- 分页测量与页面渲染共同使用固定页眉、页脚、间距和正文可用高度，不再用固定 `88` 估算；分页签名包含正文内容与标题，同长度正文刷新后也会重新分页。
- 标题排版支持左对齐、居中和隐藏，并可独立设置标题字号差值、字重和上下留白；连续滚动与分页模式使用同一套持久化配置。
- 正文首行缩进支持 `0～8` 个全角字符；两端对齐默认开启，只拉伸段落非末行，显示缩进不进入稳定字符锚点。
- 长章节分页缓存未命中时先生成首批 `8` 页或至少覆盖恢复锚点所在页，首屏可先显示；完整分页由后台小批次续算，每批处理少量段落或页面后让出事件循环。
- 超长无换行段落按 `1200` 字符切块测量，首行缩进只加在真实段落第一块，段距只加在真实段落末尾，降低单次 `TextPainter` 测量压力。
- 完整分页结果只在章节、正文、样式、尺寸和分页签名仍一致时回填，回填时按当前可见页字符位置重新定位，并缓存最近三套完整分页结果。
- 点击区域配置已接入左/中/右点击动作、长按动作、左右区域宽度和音量键翻页开关；配置进入单书缓存，旧缓存默认保持左上一页、中间菜单、右下一页。
- 分页模式点击和音量键优先翻当前页，章节边界再切章；连续模式点击和音量键优先滚动一个视口，顶部或底部再切章；长按当前支持菜单、翻页和添加书签等无选择态动作。
- 页眉页脚新增时间、电量、书名、章节名、页码和章节百分比组合；时间每分钟刷新，电量由 Route 通过平台服务低频轮询后写入 UiState。
- 阅读亮度和方向锁定已接入配置与平台服务；Android/iOS Flutter 宿主桥支持设置/恢复阅读亮度和读取电量，方向锁定使用 Flutter `SystemChrome`。
- 顶部更多菜单新增”后续能力”面板，以禁用清单登记 TTS、自动翻页、内容编辑、AI 和同步进度的前置依赖，不展示假按钮。
- 顶部更多菜单新增”单章换源”（M11 专项：[chapter_change_source](../m11/chapter_change_source/README.md)）：搜索候选来源、加载候选完整目录、模糊预选并手动确认目标章节，替换正文写入永久缓存；本地书禁用。
- 顶部更多菜单新增”下载”（M11 专项：[offline_download](../m11/offline_download/README.md)）：应用前台运行期间的章节下载队列，数据库持久化、有界并发、失败重试；无 Android 式前台服务/通知，退出应用后下载停止。

## Android 行为对照

- Android `ReadBook.saveRead()` 保存章节索引和章节字符位置；Flutter 保留这组字段，并增加章节 URL 与正文摘要锚点，目录变化时恢复更稳定。
- Android `ReadBook` 保留前、当前、后三个 `TextChapter` 并预下载更多章节；Flutter 第一批限制处理后内存缓存为三章，持久原始正文缓存七天，避免无界内存占用。
- Android `ContentProcessor` 支持更多特殊 HTML、简繁转换和内容处理规则；Flutter 第一批实现换行净化、重复标题、文本/正则替换和分块，其余处理能力保留后续映射。
- Android 页面系统支持覆盖翻页、仿真、滚动等多模式；Flutter 当前已实现连续滚动、上下分页、左右分页，以及覆盖、无动画、滑动三种基础策略，仿真和淡入等复杂动画仍待迁移。
- Flutter 覆盖模式已对照 Android `CoverPageDelegate` 的前后页层级和方向，并支持单章内跟手拖动及跨章覆盖衔接；仍需用户真机确认触摸阈值、帧率、长章节首屏速度和后台续算手感。
- Android 书签可编辑备注并可导出；Flutter P2 已支持备注编辑、删除确认和 Markdown 剪贴板导出。
- Android 自动亮度、横竖屏细项、手势区、自动翻页和 TTS 配置更完整；Flutter 当前已接入沉浸系统栏、常亮、基础亮度和基础方向锁定，更多系统栏细项仍待真机验证后继续补齐。

## 当前阻塞与后续矩阵

1. 用户尚未运行或真机验收，本阶段不能标记 `ANDROID_READY`。
2. M4 JavaScript 真实书源兼容仍为 `BLOCKED`；检测到 `jsLib` 或正文脚本标记时显示明确门禁错误，不伪装普通规则成功。
3. M3/M6 普通规则真实正文尚待用户验证，真实书源正文、分页正文和字符集结果仍依赖前置阶段验收。
4. M8.1 已把 TXT/EPUB 应用内副本和正文读取接入本协调器；UMD/MOBI/AZW/PDF 与压缩包仍是 M8.1 门禁。漫画、音频、TTS、仿真/淡入等复杂翻页、自动亮度和替换规则管理页面仍延后；书签导出和全文搜索已在阅读器内接入。
5. Dart 与 Java 正则语法及替换分组存在兼容差异；当前兼容 `$1` 与 `\1` 常见捕获替换，仍需用真实替换规则人工验证。
6. 字符锚点到当前布局的像素位置是近似投影；持久事实仍是章节 URL、字符位置和摘要，验收标准为恢复到正确章节并接近原内容。

## 用户验收步骤

AI 未运行 format、analyze、test、build、真机启动或真实网络请求。请由你执行：

1. 在 `flutter_app` 执行你需要的依赖、静态检查和 Android 构建命令，先处理任何编译错误。
2. 使用已经完成 M3/M6 验证的普通规则书源搜索一本书，进入详情获取目录并加入书架。
3. 从书架点击该书，确认 `/reader` 显示保存进度对应章节正文，不再出现 M8 占位提示。
4. 快速连续点击下一章、上一章和目录中不同章节，确认最终只显示最后选择的章节，不出现旧章覆盖或串章。
5. 滚动到章节中部，等待至少两秒后返回书架；再次打开，确认恢复到同一章节和接近原文字位置，而不是固定像素或章首。
6. 滚动后立即切章、按系统返回、切到后台，再分别重新打开，确认三条路径都保存最近字符位置。
7. 调整字号、行距、段距和边距并应用，确认正文立即重排且仍靠近原文字；切换三套配色确认文字可读。
8. 在已有替换规则数据的环境中打开规则开关，确认章末显示实际生效数量；关闭后确认从原始正文恢复，再次打开确认开关持久化。
9. 在当前位置添加两个书签，打开书签面板分别跳转，删除其中一个并确认列表实时变化。
10. 断网打开已经读过的章节，确认显示“来自正文缓存”；点击强制重试时应明确报告网络错误，不把旧缓存伪装成刷新成功。
11. 阅读当前章后切到相邻章，观察首次显示速度；快速切到非相邻章时确认当前请求优先，预加载失败不弹出当前章错误。
12. 开启常亮后停留超过系统自动锁屏时间，确认阅读器保持亮屏；关闭开关或退出阅读器后确认系统原有休眠策略恢复。
13. 确认阅读器进入时系统栏沉浸、退出后恢复 edge-to-edge；Android 返回手势只退出一次且退出前保存进度。
14. 打开已由 M8.1 导入的 TXT/EPUB，确认正文、目录、替换和恢复正常；打开未支持格式或 JavaScript 正文书源时，确认显示明确当前边界，不出现空白页、假成功或无限加载。
15. 点击正文中央显示/隐藏 P0 阅读菜单，确认菜单作为浮层覆盖正文而不是挤压正文；顶部刷新、换源、更多菜单，底部章节滑杆、上一章、目录、书签、设置、下一章均可按状态使用或禁用。
16. 拖动章节滑杆到普通章节和卷标题附近，确认松手后进入最近可阅读章节，且跳转前保存当前进度。
17. 打开“设置”，确认面板分为样式、文字、间距、系统四页；切换阅读模式、配色、字号、行高、字距、字重、斜体、段距和四周边距后点击应用，连续滚动和分页正文都能立即重排。
18. 修改设置后退出并重新进入同一本书，确认 P1 新增字段仍能恢复，且阅读位置没有固定回章首。
19. 打开目录面板，确认列表初始定位到当前章节；切换“卷标题”筛选后，卷标题不会被当作正文章节打开。
20. 从顶部更多菜单进入“搜索正文”，搜索当前章已有词，确认结果数量、上一条/下一条和点击结果都能跳到对应正文位置并保存进度。
21. 打开书签面板，编辑一条书签备注并保存，重新打开确认备注变化；删除书签时确认弹出二次确认，取消不会删除。
22. 从顶部更多菜单进入“替换统计”，确认当前章替换命中数量、替换开关状态和缓存来源展示符合当前正文处理结果。
23. 打开“设置”，切换覆盖翻页、无动画、滑动、页眉页脚、底部工具文字、文字阴影和文字下划线，确认连续滚动和分页正文按配置变化并能重新进入后恢复。
24. 在默认分页阅读模式下左右拖动或点击两侧，确认翻下一页时当前纸张向左移走并露出下一页，翻上一页时上一页从左侧覆盖回来；拖动未达到阈值会回弹，“无动画”和“滑动”仍按设置生效。
25. 从当前章末页继续左滑进入下一章，再从下一章首页右滑返回上一章，确认加载期间旧页保持可见，完成后以同一覆盖语义跨章，不出现上下列表滚动、整页转圈或正文列表刷新。
26. 从顶部更多菜单进入“后续能力”，确认依赖型能力以禁用说明展示，不会出现无响应的可点击假入口。
27. 连续打开多个长短不同的章节，确认每章第一页显示独立大号章节标题，后续页不重复该标题；正文逐行填满页面且不会为了段落换行提前留下大块空白。
28. 修改字号、行高、段距、上下边距以及页眉页脚开关，确认页面重新计算后最后一行不被裁切、页面不溢出，当前位置仍恢复到接近原字符。
