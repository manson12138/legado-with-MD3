# M6 实施记录：搜索、详情与目录

状态：`IN_PROGRESS / 普通书源闭环代码待用户验证，JavaScript 书源受 M4 BLOCKED 门禁约束`

## 已实现

- 欢迎页新增“搜索书籍”入口，稳定路由为 `/search` 和 `/book-info`。
- Search UiState、Intent、Effect、ViewModel、Route 与纯 Screen；页面不直接访问网络、规则或 DAO。
- 搜索开始、增量结果、主动停止、失败来源重试、启用书源筛选、加载、空、部分失败、全部失败和取消状态。
- 搜索开始时固定启用书源快照，最多 4 个 worker；每个书源持有独立取消令牌和 10～60 秒超时。
- ViewModel 使用递增运行编号隔离新旧搜索；即使底层取消回调迟到也不会写入新搜索状态。
- 单书源异常转换为网络、规则或 JavaScript 摘要，其他书源继续运行，不展示响应正文、Header 或 Cookie。
- 搜索结果按 Android 行为使用严格“书名 + 作者”键合并，并保留同组所有来源；精确匹配优先，其次按来源数量排序。
- 搜索历史通过 `SearchHistoryGateway` 和缓存表持久化，最近使用项置顶，最多 20 条。
- BookInfo UiState、Intent、Effect、ViewModel、Route 与纯 Screen；打开后顺序加载详情和完整分页目录。
- 详情保留搜索结果来源，合并书名、作者、封面、简介、分类、字数、最新章节和目录 URL。
- 目录沿用 M3 服务的分页循环检测、章节 URL 去重和连续索引；解析仍在 isolate 中执行。
- 未加入书架时目录保存在页面状态；点击加入书架后由 `AddBookToBookshelfUseCase` 在同一事务写入书籍和目录。
- 已在书架的书重新加载目录时，通过 `SaveBookChaptersUseCase` 整体替换目录。
- 详情页可在同次搜索合并出的同名作者来源中基础换源，换源后旧详情/目录结果由世代编号隔离。

## Android 行为对照

- Android `SearchViewModel` 以 `SearchResultKey(name, author)` 合并结果；Flutter 保持严格原始书名和作者语义，不做模糊去重。
- Android 同一书籍结果会聚合多个 origin；Flutter 使用 `BookSearchResultGroup.books` 保存每个来源，避免数据模型绑定唯一书源。
- Android 搜索停止后保留已返回结果；Flutter 主动取消同样保留部分结果，并取消当前网络令牌。
- Android 单源异常不会终止其他来源；Flutter 固定 worker 独立捕获错误并发布进度。
- Android 目录最终按显示顺序重新编号；Flutter M3 目录服务按 URL 去重后重新生成从零开始的连续索引。
- Android 完整换源会重新精确搜索、加载详情/目录并处理阅读进度；Flutter 第一批只在当前搜索已发现来源间切换，独立精确换源和进度迁移登记为后续能力。

## 用户验收步骤

AI 未运行网络请求、测试、analyze、format 或 build。请由你执行：

1. 在 `flutter_app` 执行依赖获取、静态检查和你需要的构建命令。
2. 启动应用，先在书源管理中确认至少一个普通规则书源已启用。
3. 从欢迎页点击“搜索书籍”，输入固定关键字并开始搜索。
4. 确认进度不会超过启用书源数，单源失败可展开查看且不会终止其他书源。
5. 搜索中点击停止，确认已返回结果保留；立刻提交另一个关键字，确认旧结果不会混入。
6. 选择部分书源搜索，再用“重试失败项”确认只运行失败来源。
7. 退出再进入搜索页，确认最近关键字仍存在，清空后再次进入不再出现。
8. 打开一个普通规则结果，确认详情字段和完整目录可加载，章节索引从 1 显示且无重复 URL 对应项。
9. 若结果显示多个来源，使用右上角换源，确认详情和目录切换到新来源。
10. 点击“加入书架”，确认立即显示“已在书架”；检查数据库中书籍和章节属于同一 `bookUrl`，章节索引连续且唯一。
11. 按 `m04/05_collection_validation_samples.md` 的固定样本验证 JavaScript 书源；先验证 S2、S3 的搜索、详情、目录和正文，再确认 S4、S5 分别报告同步网络语义和 JVM 类兼容边界。

用户提供普通规则闭环及 M4 JavaScript 结果前，M6 保持 `IN_PROGRESS`。
