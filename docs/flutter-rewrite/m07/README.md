# M7 实施记录：书架

状态：`IN_PROGRESS / 第一批书架代码待用户验证，目录刷新受 M4/M6 真实书源门禁约束`

## 已实现

- 欢迎页新增“打开书架”入口，稳定路由为 `/bookshelf`。
- Bookshelf UiState、Intent、Effect、ViewModel、Route 和纯 Screen；列表与网格共享同一业务状态。
- 书架直接观察 `BookshelfGateway.watchBookshelf()`，M6 加入书架或后续阅读进度写入会触发数据库流重建页面。
- `BookshelfBookItem` 独立承载显示字段，未修改 `Book` 实体语义；未读数使用 Android 公式 `totalChapterNum - durChapterIndex - 1`，最小为 0。
- 列表和响应式网格均使用 `bookUrl` 作为稳定 key；支持书名、作者、来源和分类搜索。
- 支持最近阅读、最新更新、书名、手动顺序、最近活动和作者六种 Android 对齐排序字段，支持正序/倒序，并以 `bookUrl` 稳定次级排序。
- 固定提供全部、未分组、未读和阅读中系统分组；观察数据库用户分组并按分组 order、groupId 稳定排序。
- 长按进入选择模式，点击切换选择，全选只作用于当前可见项；返回优先关闭对话框和退出选择模式。
- 支持批量创建用户分组、移动到已有分组、清除用户分组；写入通过 UseCase/Repository 完成。
- 批量删除先展示明确影响范围，再由单一事务删除书籍；章节依赖 SQLite 外键级联，不由 UI 重复删除。
- `BookshelfRefreshCoordinator` 使用最多 3 个 worker、每书独立取消令牌和 60 秒超时；单书失败不终止其他任务。
- 刷新结果通过 `AddBookToBookshelfUseCase` 原子替换书籍与完整目录，更新目录总数、最新章节、更新时间和未读显示。
- 刷新使用世代编号隔离取消后的旧事件，书架流变化时自动剔除已删除书籍的选择状态。
- 公共 `BookCover` 支持网络封面、本地路径、缺失和加载失败占位；网络图片复用 Flutter 默认内存 ImageCache，不阻塞业务状态。
- 书籍菜单可打开 M6 详情；点击书籍发出带稳定 `bookUrl` 的阅读器导航 Effect。

## Android 行为对照

- Android 默认排序为最近阅读倒序；Flutter 使用相同默认值，并实现 Android `BookshelfRepository.sortBooks` 的六个字段。
- Android 用户分组使用正数位值；Flutter 创建分组时从最低未使用二进制位开始分配，书籍分组仍保存为整数位值。
- Android 书架支持多分组位掩码；Flutter 第一批“移动分组”使用替换语义，一本书只保留目标用户分组，后续再增加追加/移除多分组。
- Android 刷新队列使用配置线程数；Flutter 第一批固定最多 3 个 worker，避免无界 Future。
- Android 删除本地书时可选择是否删除原文件，并执行书源回调和正文缓存清理；Flutter 当前数据库不保存正文缓存，删除仅处理 Flutter 书籍与级联目录，不删除本地原始文件。
- Android 支持文件夹式分组、分组封面、拖拽手动排序和高级管理；Flutter 第一批提供分组选择、新建和移动，相关高级 UI 延后。

## 用户验收步骤

AI 未运行数据库、网络请求、UI、format、analyze、test 或 build。请由你执行：

1. 在 `flutter_app` 执行依赖获取、静态检查和需要的构建命令。
2. 先按 M6 路径将至少两本普通规则书籍加入书架，再从欢迎页打开书架。
3. 确认新增书籍无需重进页面即可出现；封面缺失或失败时显示占位，不出现布局跳动。
4. 切换列表和网格，确认两种布局的分组、排序、搜索和选择状态一致。
5. 依次检查六种排序和正倒序；相同主排序值的书籍顺序应保持稳定。
6. 检查全部、未分组、未读、阅读中分组数量和内容；阅读进度变化后重新进入确认未读数变化。
7. 长按一本书进入选择模式，点击其他书切换选择，按返回确认先退出选择而不是直接离开页面。
8. 选择多本书，新建用户分组并移动；确认分组立即出现且数量正确，再执行清除分组。
9. 刷新当前列表，确认进度、成功和单书失败统计；刷新中点击停止，确认不再调度后续书籍且旧结果不污染下一次刷新。
10. 只选部分书籍刷新，确认未选书籍不进入更新集合；JavaScript 书源当前应显示明确 M4 门禁错误。
11. 选择书籍并删除，确认先显示影响范围；删除后书籍立即消失，章节表不存在该 `bookUrl` 的孤立目录，本地原文件未删除。
12. 点击详情图标确认进入 M6 详情；点击书籍确认进入 `/reader` 并由 M8 打开该书保存进度对应章节。
