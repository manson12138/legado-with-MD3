# M9 缺陷与回归记录

## 严重级别

- `P0`：启动崩溃、数据破坏、安全问题或核心路径完全不可用。
- `P1`：搜索、详情、加书架、阅读、恢复或强制格式验收被阻断。
- `P2`：存在可用绕行方案，但行为明显偏离 Android 基线。
- `P3`：不阻断使用的视觉、提示或低频问题。

## 新问题模板

| 字段 | 必填内容 |
|---|---|
| ID / 严重级别 | 例如 `M09-P1-006 / P1` |
| 所属阶段 | M1～M8.1 中真实责任阶段 |
| 环境 | 设备、系统、App 提交、安装类型、网络 |
| 输入 | 书源样本编号、关键词、书籍章节或本地格式；不粘贴敏感正文 |
| 复现步骤 | 从稳定初始状态开始的最短步骤 |
| 预期 | Android 基线或阶段文档定义的行为 |
| 实际 | 可观察结果，不写“不能用”这种模糊描述 |
| 日志范围 | 发生时间前后必要堆栈，已脱敏 |
| 修复文件 | 实际修改路径 |
| 回归范围 | 直接路径、相邻功能、重启恢复和异常路径 |
| 结论 | PASS / FAIL / BLOCKED |
