## 阶段目标

让用户能够把真实书源放入新 App，并完成查看、启停、分组、编辑、删除和基础调试，为搜索提供可管理的数据来源。

## 功能范围

- JSON 文件导入。
- 文本/剪贴板导入。
- 二维码导入接口。
- 书源列表、搜索和筛选。
- 启用/停用。
- 分组。
- 编辑与保存。
- 删除。
- 基础调试与错误查看。
- 登录、验证码和 Cookie 入口（样本需要时）。

## 执行步骤

### 1. 行为盘点

- 记录原页面入口、菜单、长按、批量选择、导入冲突和返回行为。
- 记录 BookSource 字段编辑规则和必填校验。
- 记录导入同 URL 书源时覆盖、跳过或合并行为。

### 2. Contract

- UiState 包含加载、列表、筛选、选择模式、对话框和导入摘要。
- Intent 覆盖导入、启停、分组、编辑、选择、删除、调试和返回。
- Effect 处理文件选择、二维码、WebView 登录、Toast 和导航。

### 3. 导入流程

- 外部文件先解析到导入 DTO，再校验和转换 Entity。
- 显示成功、失败、重复、覆盖和跳过数量。
- 批量写入使用事务。
- 单条错误不应让全部导入静默失败，具体策略必须与原行为或用户决策一致。

### 4. 列表与编辑

- 长列表稳定 key 使用书源 URL 或明确业务 ID。
- 支持单项和批量启停/删除。
- 编辑页字段分组与原功能一致，视觉采用统一 Flutter 规范。
- 未保存离开时按原行为提示。

### 5. 调试与登录

- 调试结果显示网络、规则和 JavaScript 错误分类。
- 登录和验证码通过平台边界调用 WebView。
- Cookie 回写统一 Cookie 管理器。

## 注意点

- 书源 JSON 可能是单对象、数组、转义字符串或远程内容。
- BookSource URL 常是唯一标识，修改 URL 会影响覆盖语义。
- 列表启用状态变化会直接影响搜索数据源。
- 编辑器输入包含大量反斜杠、正则和 JavaScript，不能错误转义。
- 删除书源是否影响书架已有书籍必须明确，不能顺便级联删除。
- 二维码内容可能是 URL 或直接 JSON，必须验证来源和大小。

## 完成标准

2. 重复、无效、部分失败和覆盖行为有明确结果。
3. 列表可查看、搜索、启停、分组、编辑和删除。
4. 编辑保存后搜索层能读取最新配置。
5. 登录/验证码所需 Cookie 能回到统一存储。
6. UI 包含加载、空、错误、选择和确认状态。
7. 用户执行导入和管理验收后确认；未确认前不标记完成。
