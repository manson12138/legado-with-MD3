> 当前状态：`IN_PROGRESS / 普通书源闭环代码已实现，待用户运行验证；JavaScript 书源受 M4 BLOCKED 门禁约束`。实施记录见 `../m06/README.md`。

## AI 执行规则（必须先读）

1. 必须确认 M5 能提供启用书源，并阅读总方案及 M0～M5 文档。
2. 必须检查 Android 搜索、详情、目录、换源及相关 ViewModel/Model/UseCase。
3. 多书源并发必须有限制、可取消、可展示单源错误，禁止无界 Future。
4. 页面只消费状态，不直接调用规则引擎或 DAO。
5. 搜索结果去重、详情合并、目录排序和 URL 解析必须对照 Android 行为。
6. 导航与一次性提示使用 Effect。
7. 所有新增类、方法和变量必须有中文注释；禁止 Dart `!`。
8. AI 不运行搜索请求、测试、analyze 或 build。

## 阶段目标

打通从启用书源到搜索结果、书籍详情、章节目录和加入书架的业务链路。

## 执行步骤

### 1. 搜索行为盘点

- 记录关键字处理、搜索历史、书源选择、并发数、取消、超时和进度显示。
- 记录结果去重键、排序、来源信息和失败统计。
- 记录重新搜索、返回和旋转/重建后的状态恢复。

### 2. Search Contract

- UiState 包含关键字、状态、结果、书源进度、错误摘要和筛选。
- Intent 包含输入、开始、取消、重试、选择书源、打开详情和返回。
- Effect 包含导航、Toast 和系统行为。

### 3. 搜索协调器

- 从 Repository 获取当前启用书源快照。
- 使用受控并发执行搜索。
- 支持单书源超时和整体取消。
- 结果按稳定规则增量合并，页面不负责去重。

### 4. 详情解析

- 保存搜索结果携带的来源信息。
- 解析封面、作者、简介、分类、最新章节等字段。
- 处理相对 URL、重定向最终 URL 和缺失字段。
- 加入书架通过 UseCase 完成事务写入。

### 5. 目录解析

- 解析章节列表、章节 URL、标题、索引、卷信息和更新时间。
- 处理章节反序、重复、分页目录和缺失 URL。
- 写入目录时保证同一本书的一致性。

### 6. 基础换源边界

- 建立同名书或相似书的换源入口。
- 第一批可以只提供基础选择，但数据模型不能绑定唯一原书源实现。

## 注意点

- 搜索取消后旧任务结果不得污染新搜索。
- 相同书名作者不一定是同一本书，去重规则要对照原项目。
- 单一书源失败不应自动终止全部搜索。
- 详情 URL 与目录 URL 可能经过脚本改写。
- 目录可能非常大，解析和写入不得阻塞 UI。
- 加入书架后详情页面状态应立即一致，不能等待重新进入。

## 完成标准

1. 普通规则和 JavaScript 书源至少各有一份可搜索。
2. 搜索可开始、增量展示、取消、重试并显示失败摘要。
3. 可从结果打开详情并获取完整目录。
4. 可加入书架，数据库中书籍和目录关系正确。
5. 旧搜索不会污染新搜索。
6. 页面覆盖加载、空、部分成功、全部失败和取消状态。
7. 用户按真实书源路径验收并确认。
