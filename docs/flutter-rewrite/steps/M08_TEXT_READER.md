## AI 执行规则（必须先读）

1. 必须确认 M7 可提供稳定阅读入口，并阅读总方案及 M0～M7 文档。
2. 必须检查 Android `ReadBookActivity`、`ReadBook`、页面实现、配置、目录、书签、替换规则和缓存逻辑。
3. 阅读位置必须使用稳定内容定位，不得只保存当前滚动像素。
4. 第一批允许只实现上下滚动，但架构必须保留覆盖翻页、动画和 TTS 扩展能力。
5. 正文获取、缓存、替换和进度保存不得写入 Widget。
6. UI isolate 不得执行大正文解析、复杂正则或批量排版。
7. 系统栏、屏幕常亮、亮度等使用平台接口/Effect，业务状态保持跨平台。
8. 所有新增类、方法和变量必须有中文注释；禁止 Dart `!`；AI 不运行任何检查。

## 阶段目标

完成第一批可用文本阅读器，使用户可以读取章节、切换章节、调整基础显示、保存并恢复阅读位置。

## 第一批范围

- 正文加载与缓存。
- 上下滚动。
- 上一章/下一章。
- 目录跳转。
- 阅读进度保存与恢复。
- 字号、行距、段距、边距。
- 背景与文字颜色。
- 书签。
- 替换规则。
- 前后章节预加载。
- 错误重试。
- 屏幕常亮与系统栏。

## 执行步骤

### 1. 原行为盘点

- 记录阅读入口参数、章节定位、退出保存、翻页方式和返回行为。
- 记录正文净化、替换规则、缓存、刷新和章节错误处理。
- 记录字体、主题、亮度、状态栏、横竖屏和手势。

### 2. 阅读领域模型

- 定义 `ReadBook` 对应协调器职责。
- 定义当前书、章节、正文块、位置锚点、显示配置和加载状态。
- 位置锚点至少包含章节稳定标识和章节内可恢复位置。
- UI 滚动状态与持久阅读进度分离。

### 3. 正文管线

- 章节选择 → 缓存查询 → 网络/规则获取 → 解码 → 净化 → 替换 → 分块 → 展示。
- 每一步有明确错误类型和取消行为。
- 新章节请求开始后旧请求不得覆盖当前章节。

### 4. 阅读器 Contract

- UiState 包含书籍、章节、正文块、进度、配置、菜单、加载和错误。
- Intent 覆盖点击区域、滚动进度、章节切换、目录、书签、配置和重试。
- Effect 处理系统栏、屏幕常亮、亮度、导航和提示。

### 5. 滚动阅读

- 使用惰性列表/合适文本分块，避免单个超大 Text 导致内存和布局问题。
- 稳定 key 由章节与内容块标识组成。
- 章节切换时正确释放旧资源并恢复目标位置。

### 6. 进度保存

- 定义节流保存、章节切换保存、后台保存和正常退出保存。
- 再次打开时先恢复章节，再恢复章节内位置。
- 字体或宽度变化后通过内容锚点重新定位。

### 7. 预加载和错误恢复

- 当前章节完成后预加载前后章节，限制并发和缓存大小。
- 当前请求优先于预加载。
- 预加载失败不打断当前阅读。

## 注意点

- 原 Android 阅读器仍是复杂 View 系统，不能只参考一个 Activity 文件。
- 不同字体和屏幕宽度会改变分页/滚动位置，像素偏移不可作为唯一进度。
- 替换规则可能非常耗时或产生空正文，必须区分真实空内容和处理错误。
- 快速连续切章需要取消旧任务。
- 横竖屏、系统字体缩放和安全区域变化会触发布局重算。
- 第一批不实现动画，不代表可以把正文状态绑定死在滚动列表。

## 完成标准

1. 可从书架打开一本真实书并显示当前章节正文。
2. 可连续切换前后章节且快速操作不会串章。
3. 可退出并再次打开，恢复到正确章节和接近原内容位置。
4. 字号、行距、边距和颜色调整可立即生效。
5. 目录跳转、书签、替换规则和错误重试可用。
6. 预加载不会阻塞当前章节，也不会无界占用内存。
7. 用户在真实设备完成人工阅读验收。
