## AI 执行规则（必须先读）

1. 必须确认 M6 能稳定加入书架，并阅读总方案及 M0～M6 文档。
2. 必须检查 Android 书架、分组、管理、更新目录、封面和选择模式实现。
3. 书架页面必须使用 State/Intent/Effect，列表状态不得分散在 Widget 和 ViewModel 两处。
4. 删除、分组、排序和批量操作必须通过 UseCase/Repository。
5. 长列表必须使用稳定书籍 ID/URL 作为 key，不得使用数组下标。
6. 不得为了 UI 方便修改核心实体含义；显示模型可单独定义。
7. 所有新增类、方法和变量必须有中文注释；禁止 Dart `!`。
8. AI 不运行数据库、UI、analyze、test 或 build。

## 阶段目标

提供第一批可用书架，使用户可以查看已加入书籍、分组排序、更新目录、管理选择并进入阅读器。

## 执行步骤

### 1. 行为盘点

- 记录列表/网格、分组、排序、未读、封面、刷新、长按选择和返回行为。
- 记录删除时正文缓存、目录、书签和阅读进度的处理。
- 记录目录更新触发方式、并发和错误展示。

### 2. Contract

- UiState 包含布局模式、分组、排序、书籍、刷新、选择集合、对话框和错误摘要。
- Intent 覆盖打开、刷新、布局切换、分组、排序、选择、批量操作、删除和返回。
- Effect 处理导航阅读器、详情、Toast 和系统操作。

### 3. 列表数据

- Repository 提供可观察书架流。
- ViewModel 将实体映射为稳定显示模型。
- 封面加载失败、空封面和缓存策略由公共 `BookCover` 处理。
- 大量数据变化时避免整页无意义重建。

### 4. 分组与排序

- 对照 Android 的默认分组和排序字段。
- 保证同序值时有稳定次级排序。
- 分组切换不应丢失合理的滚动或选择状态。

### 5. 更新目录

- 使用受控并发。
- 单书失败不影响其他书更新。
- 结果更新未读数、最新章节和更新时间。
- 用户取消后停止继续调度。

### 6. 选择与删除

- 长按进入选择模式，返回优先退出选择模式。
- 批量操作只作用于稳定 ID 集合。
- 删除前显示影响范围并按既定事务执行。

## 注意点

- 书架数据可能在更新目录时持续变化，选择集合必须自动剔除已删除项。
- 封面加载不应阻塞列表滚动。
- 网格和列表应共享同一业务状态，而不是两个 ViewModel。
- 返回行为要区分选择模式、对话框和页面退出。
- 书籍打开后阅读进度更新应反映到书架。
- 删除级联必须与数据层定义一致，不得在 UI 手动多次删除。
