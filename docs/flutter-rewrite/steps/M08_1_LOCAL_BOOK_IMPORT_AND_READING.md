# M8.1：本地书籍导入与阅读

> 当前状态：`IN_PROGRESS / TXT、EPUB、UMD 与 PDF 主链路已实现，待用户运行；MOBI 家族和压缩容器仍是阶段门禁`。

## AI 执行规则（必须先读）

1. 必须确认 M8 阅读器代码边界已经稳定，并完整阅读总方案、M0～M8、M8 实施记录和本文件。
2. 必须检查 Android `ImportBookScreen`、`ImportBookViewModel`、`LocalBook`、全部本地解析器、文件选择、压缩包、删除和阅读调用路径，不能只实现文件选择按钮。
3. Android 支持的本地书格式不得静默缩减；不能实现的格式必须返回明确 `unsupported`，并保留在本阶段门禁中。
4. 文件选择、权限和安全作用域 URL 属于平台边界；书籍识别、分章、元数据、正文和进度模型优先使用 Dart。
5. 导入后必须保存可长期重开的应用内文件或持久文件授权，不能只保存一次性 picker 临时路径。
6. 不得在 UI isolate 读取整本大文件、解压大压缩包、执行目录正则或解析 EPUB/MOBI/PDF。
7. 本地文件身份、同名文件、重复导入、原文件变化和删除语义必须明确，不能把文件名当唯一主键。
8. M8 的阅读进度仍使用稳定章节标识、章节内字符或页面位置；禁止只保存滚动像素。
9. 所有新增类、方法和变量必须有详细中文注释；禁止 Dart、Swift 强制空值断言和 Kotlin `!!`。
10. AI 不运行格式化、检查、测试、构建或真机操作；用户验证前状态只能是 `IN_PROGRESS`。

## 阶段目标

完成从系统文件入口到书架和阅读器的本地书闭环：用户能够选择或接收本地书文件，识别书籍信息和目录，加入书架，读取正文或页面，退出后再次恢复阅读位置。

M8.1 完成后，M9 Android 第一批验收必须同时覆盖网络书源书籍和本地书籍。

## Android 格式基线

| 类型 | Android 入口 | Android 解析器 | M8.1 Flutter 行为 |
|---|---|---|---|
| TXT | `LocalBook` | `TextFile.kt` | 编码识别、自定义编码、目录规则、分章、正文字符锚点 |
| EPUB | `LocalBook` | `EpubFile.kt` | OPF/NCX/Nav 目录、HTML 正文、图片和封面、fragment 边界 |
| UMD | `LocalBook` | `UmdFile.kt` | 元数据、目录、正文和封面 |
| MOBI | `LocalBook` | `MobiFile.kt` | MOBI 元数据、目录、正文和资源 |
| AZW/AZW3 | `LocalBook` | `MobiFile.kt` | 按 Android 当前 MOBI 家族行为解析；受 DRM 文件必须明确拒绝 |
| PDF | `LocalBook` | `PdfFile.kt` | 页面目录和页面渲染；进度使用页码与页内位置，不伪装成文本字符正文 |
| ZIP/RAR/7Z | `ImportBookViewModel`、`LocalBook.importArchiveFile` | `ArchiveUtils` | 作为导入容器列出支持书籍，防路径穿越、压缩炸弹和无界解压 |

- 漫画压缩包阅读模式；压缩包在本阶段仅作为书籍导入容器。
- OCR、PDF 文本重排、PDF 批注和密码破解。
- Android 与 iOS 各写一套重复的书籍解析业务。

## 执行步骤

### 1. 完整行为盘点

- 记录 `MainNavGraph` 本地书入口、系统文件关联、分享/打开入口和返回行为。
- 记录 `ImportBookUiState`、Intent、Effect、文件夹导航、搜索、排序、多选、反选和批量加入书架。
- 记录默认书籍目录、扫描目录、自动同步、持久文件权限和外部文件变化行为。
- 记录压缩包条目选择、批量导入、失败统计和临时文件清理。
- 记录导入后直接打开、加入书架标记、重复导入更新、删除记录与删除文件的差异。

### 2. 文件平台边界

- 定义 `LocalBookFilePicker`、`LocalBookFileAccess` 和平台错误模型。
- Android 使用 Storage Access Framework/系统文件选择器；持久 URI 权限失败时复制到应用目录。
- iOS 使用 Document Picker；需要长期读取时复制到应用沙盒，安全作用域访问必须成对开始和结束。
- 平台层只提供选取、流式读取、复制、元数据和删除能力，不解析章节或持有阅读状态。
- 文件句柄、临时文件和安全作用域授权在成功、失败、取消三条路径全部释放。

### 3. 稳定文件身份与存储策略

- 定义本地书文件记录：原始显示名、应用内相对路径、格式、大小、修改时间、内容指纹和可选外部引用。
- `bookUrl` 使用稳定应用内身份，不直接依赖会失效的临时绝对路径。
- 同一文件重复导入应更新既有书籍和目录；同名但内容不同的文件必须允许并存或明确冲突选择。
- 文件修改后保留书架元数据和用户进度，并通过章节标题/地址映射恢复最接近章节。
- 数据库书籍、章节、文件副本和封面写入需要补偿或事务边界，避免留下半导入记录。

### 4. 统一本地书解析接口

- 定义 `LocalBookParser`：格式探测、元数据、目录、正文块/页面、封面和资源读取。
- 扩展名只作为候选，必须结合文件签名或容器结构确认格式。
- 解析结果复用 `Book`、`BookChapter` 和 M8 阅读锚点，不把第三方库对象暴露到领域/UI。
- 解析任务提供取消和阶段进度：复制、探测、元数据、目录、封面、持久化。
- 每种格式使用独立实现和固定样本，统一由 `LocalBookParserRegistry` 选择。

### 5. TXT 解析

- 读取 BOM，并在无 BOM 时对 UTF-8、GB18030/GBK、Big5 等候选编码做受控探测。
- 用户可手动覆盖编码；切换编码后清除旧目录/正文缓存并重新解析。
- 对照 Android TXT 目录规则，支持默认章节正则和用户选择的目录规则。
- 大 TXT 使用流式扫描或 isolate 分块，不把整本文件同时复制多份到内存。
- 章节保存稳定文件范围和标题；正文读取只读取目标范围，并接入 M8 替换与字符锚点。
- 无章节匹配时创建单章，而不是把合法文本误判为导入失败。

### 6. EPUB 解析

- 校验 ZIP 容器、`container.xml`、OPF、manifest、spine、NCX 或 EPUB3 nav。
- 解析书名、作者、封面、目录层级、线性阅读顺序和资源基准地址。
- HTML 正文净化后转换为文本块、图片块和必要样式语义，不将脚本带入阅读器执行。
- 相对图片、SVG、字体和锚点资源通过受控应用内资源 URI 访问。
- 章节稳定标识使用 spine href/fragment，不只使用列表索引。
- 损坏目录可按 spine 降级；缺失正文、越界资源和非法路径必须明确报错。

### 7. UMD 与 MOBI 家族解析

- 对照 `UmdFile.kt`、`MobiFile.kt` 和 `modules:book` 的实际字段、章节边界、封面及编码行为建立样本。
- 优先采用可在 Android/iOS 共用的 Dart 解析；引入第三方库前记录维护状态、格式覆盖、许可证和内存风险。
- MOBI、AZW、AZW3 先识别 DRM/加密标志；受保护文件返回明确不支持，不尝试绕过。
- 内嵌 HTML、图片和资源使用与 EPUB 相同的安全内容块边界。

### 8. PDF 页面阅读

- PDF 不进入普通 `ReaderChapterContent.text` 管线，定义独立 `LocalDocumentPage`/页面内容提供器。
- 目录优先读取 PDF outline；无 outline 时以页面或受控页组生成目录。
- 页面渲染、缩放和预加载通过跨平台插件或最小原生接口，Dart 保存页码、页内比例和目录状态。
- 页面位图设置严格尺寸和 LRU 上限；离开页面及时释放。
- 加密但可由用户密码打开的 PDF 与不支持/损坏 PDF 分开建模；密码交互若本阶段不实现必须明确登记。

### 9. 压缩包安全导入

- ZIP、RAR、7Z 仅列出匹配支持格式的文件条目。
- 拒绝绝对路径、`..` 路径穿越、符号链接逃逸和异常压缩比。
- 设置条目数、单文件大小、解压后总大小和临时目录上限。
- 用户选择条目后流式复制到应用书籍目录，再走普通格式解析。
- 成功、失败和取消后清理临时文件；压缩包原文件的保留策略不由 UI 隐式决定。

### 10. 导入 Contract 与 UI

- 新建 `LocalBookImportContract`、ViewModel、Route 和无状态 Screen。
- UiState 包含当前位置、候选文件、格式、大小、书架状态、选择、排序、搜索、扫描/导入进度和逐文件错误。
- Intent 覆盖选择文件、进入目录、返回层级、扫描、搜索、排序、选择、反选、导入、取消和重试。
- Effect 负责文件选择、权限、打开阅读器、删除确认和用户消息。
- 批量导入单文件失败不得回滚其他成功文件；最终展示成功、重复更新、跳过和失败统计。

### 11. 书架与阅读器接线

- 欢迎页/书架新增明确“导入本地书”入口。
- 导入事务写入 `origin = loc_book`、本地类型位、书籍元数据和完整目录。
- M7 书架封面、分组、排序、删除和文件丢失状态支持本地书。
- M8 `ReadBookCoordinator` 按本地书类型走 `LocalBookContentService`，不再返回 M08 暂不支持错误。
- TXT/EPUB/UMD/MOBI 使用文本或混合内容阅读；PDF 路由到页面阅读组件，但共享菜单、目录、书签和系统 Effect。
- 相邻章预加载只读取受控范围；不把整本大文件预读进内存。

### 12. 进度、书签、替换与文件更新

- 文本类进度使用稳定章节标识、章节内字符位置和正文摘要。
- EPUB/MOBI 章节变化后优先匹配 href/spine/标题；TXT 重新分章后使用标题和邻近正文恢复。
- PDF 进度至少保存文件稳定身份、页码和页内比例。
- 文本类本地书使用 M8 替换规则；PDF 不执行正文替换。
- 文件内容更新后重新解析目录并迁移进度；失败时保留旧数据库事实并提示用户。
- 文件丢失时书架保留可诊断状态，允许重新选择文件绑定；不能直接把书籍静默删除。

### 13. 删除和生命周期

- 删除书架记录与删除应用内书籍副本必须是两个明确选项。
- 外部原文件默认不删除；只有平台允许且用户明确确认时才请求删除。
- 删除时同步处理章节、封面、解析缓存、临时文件和持久授权引用。
- App 后台、系统回收或取消导入时保存可恢复状态或执行完整清理，不留下永久 loading。

### 14. 样本和性能基线

- 每种格式至少准备：最小合法、真实中文书、超大文件、损坏文件各一份。
- EPUB/MOBI 增加图片、目录缺失、相对路径和非法路径样本。
- TXT 增加 UTF-8、GB18030/GBK、Big5、无章节和超长章节样本。
- PDF 增加有/无 outline、大页面、图片型、密码或加密样本。
- 压缩包增加多书、嵌套目录、路径穿越和异常压缩比样本。
- 记录导入耗时、峰值内存、首章打开时间、连续切章和退出后资源回收结果。

## 错误模型

至少区分：

- 用户取消选择。
- 临时权限失效或安全作用域访问失败。
- 文件不存在、为空或读取失败。
- 格式不支持、扩展名与真实格式不一致。
- 编码无法确认。
- 容器、目录、正文或资源损坏。
- DRM/加密不支持。
- 压缩包安全限制拒绝。
- 存储空间不足或复制失败。
- 解析取消、超时或后台 isolate 失败。
- 数据库写入失败和补偿清理失败。

任何错误都不能转换为空书、空目录或永久加载状态。

## 完成标准

1. 用户可通过系统文件选择器导入 Android 支持的全部本地书格式，或对受保护文件得到明确不支持原因。
2. 导入成功后书籍立即出现在书架，封面、名称、作者和目录与文件事实一致。
3. TXT、EPUB、UMD、MOBI/AZW/AZW3 可打开正文、切章、使用目录、书签和替换规则。
4. PDF 可按页面阅读、缩放、跳转并保存恢复页码，不借用文本字符模型造假。
5. ZIP/RAR/7Z 可安全选择支持条目，恶意或异常压缩包不会越界写入或无界占用资源。
6. 同一文件重复导入会更新而不是制造不可解释副本；同名不同内容具有明确冲突行为。
7. 关闭并重启 App 后，文件仍可访问，目录和阅读位置正确恢复。
8. 文件变化、丢失、权限失效、存储不足和解析损坏均有可重试的明确反馈。
9. 大文件解析不阻塞 UI isolate，取消后旧任务不能写入书架或覆盖新结果。
10. 用户在真实 Android 设备完成全格式人工验收；未验证前只能保持 `IN_PROGRESS`。
