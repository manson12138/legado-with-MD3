## 核心验收路径

1. 安装独立 Flutter App，确认原 Android App 仍可共存。
2. 首次启动，确认无旧 App 数据依赖。
3. 导入普通规则书源。
4. 导入 JavaScript 书源。
5. 启用、停用、编辑并重新启用书源。
6. 搜索一本书，观察部分结果和失败提示。
7. 打开详情并获取目录。
8. 加入书架。
9. 从书架打开阅读器。
10. 阅读正文并切换章节。
11. 修改字号、行距和颜色。
12. 添加书签并使用目录跳转。
13. 退出到书架并关闭 App。
14. 重新启动并打开同一本书。
15. 确认章节和阅读位置恢复。

本地书分支同样属于 M9 核心验收：

1. 通过系统文件选择器分别导入 TXT、EPUB、UMD、MOBI/AZW3 和 PDF 样本。
2. 从 ZIP/RAR/7Z 样本中选择至少一本支持书籍导入。
3. 确认书籍、封面和目录出现在书架，并可打开对应文本或页面阅读器。
4. 切章或翻页、添加书签、退出并关闭 App。
5. 再次启动后确认本地文件仍可访问，并恢复正确章节/页面和接近原位置。
6. 验证损坏文件、DRM 文件、恶意压缩包、权限失效和文件丢失均有明确错误。

## 异常路径

- 无网络。
- 书源返回 HTTP 错误。
- 普通规则无匹配。
- JavaScript 抛异常。
- JavaScript 超时。
- 目录为空。
- 正文为空。
- 快速重复搜索。
- 搜索中取消。
- 快速连续切章。
- 删除当前书源后打开书架书籍。
- 本地书文件丢失、权限撤销或内容发生变化。
- 本地书格式损坏、DRM 不支持和压缩包安全限制拒绝。
- 数据库写入失败或磁盘空间不足的可诊断行为。

## 注意点

- 只在模拟器启动不等于真实 Android 可用。
- 只验证一个简单普通规则书源不等于 JavaScript 兼容完成。
- UI 看起来正确不等于进度、数据库和取消行为正确。
- 日志不能包含 Cookie、账号和正文。
- 验收发现的问题应回到所属阶段修复并重新走受影响路径。
